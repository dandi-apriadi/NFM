# Contributing to NFM

Thank you for your interest in the Neural Fragment Mesh (NFM)! We are building a sovereign L1 AI-Blockchain and we need your help.

## How Can I Contribute?

### Reporting Bugs
- Ensure the bug was not already reported by searching on GitHub [Issues](https://github.com/dandi-apriadi/NFM/issues).
- If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/dandi-apriadi/NFM/issues/new).

### Suggesting Enhancements
- Check the [Roadmap](./docs/implementation_roadmap.md) to see if it's already planned.
- Open a new issue with the "feature request" tag.

### Code Contributions
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/cool-feature`).
3. Commit your changes (`git commit -m 'Add cool feature'`).
4. Push to the branch (`git push origin feature/cool-feature`).
5. Open a Pull Request.

## Coding Standards
- **Rust**: Follow `fmt` and `clippy`. Use `cargo fmt --all` before committing.
- **Documentation**: All public APIs in `core/blockchain` should be documented.
- **Tests**: Ensure `cargo test` passes before pushing.

## AI Safety & Ethics
When contributing to the AI Engine or Core Brain modules, please adhere to our AI Safety guidelines as outlined in our [Blueprint](./blueprint.txt).
